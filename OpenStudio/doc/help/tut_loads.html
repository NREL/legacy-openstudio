Zone Loads


Internal Gains

  • Internal gains include people, lights, equipment, outdoor air for ventilation, and infiltration. Unlike the Object Info dialog, this dialog does not allow user to view internal gains in current zones. It only allows the user to specify values that will be written to the file.
  • Internal gains are modeled using the input objects 'People', 'Lights', 'ElectricEquipment', 'GasEquipment', 'ZoneVentilation:DesignFlowRate', and 'ZoneInfiltration:DesignFlowRate'. Most fields of these objects are defaulted. Look at the InputOutputReference and Energy+.idd for a full list of all fields.
  • A set of internal gains and schedules for a typical office building have been added for reference only. The schedules are present in the NewTemplate.idf file but will not be available in all existing IDF files. You can reset the internal gains dialog to these values with the "Reset to Default Office" button.
  • No IDF objects will be added for internal gains with values less than or equal to zero.
  • Applying internal gains to selected zones removes existing objects from the selected zones.
  • Lights objects are required in order to add Daylighting:Controls to a zone.
  • Renaming zones after applying internal gains can cause problems. However, you can always fix these by hand in the IDF.
  • You can get the last Zone Loads Report in Comma Separated Value (CSV) format to verify the operations that have been performed. You can also save a copy of the IDF before and after the operation and then run a DIFF using a file comparison tool.

Ideal Loads Air System

  • EnergyPlus simulations may experience problems if internal gains are present but no system is included to remove energy from zones with internal gains.
  • For models which are starting from scratch, HVACTemplate:Zone:IdealLoadsAirSystem gives the quickest path for load calculations using EnergyPlus.
  • Adding HVACTemplate:Zone:IdealLoadsAirSystem to selected zones removes existing objects of this type from the selected zones.
  • A HVACTemplate:Thermostat is referenced by each HVACTemplate:Zone:IdealLoadsAirSystem to specify heating and cooling setpoints.
  • You can get the last Zone Loads Report in Comma Separated Value (CSV) format to verify the operations that have been performed. You can also save a copy of the IDF before and after the operation and then run a DIFF using a file comparison tool.

Saved Preferences

  • The initial values for internal gains correspond to a general office building. The initial schedules are those included in NewTemplate.idf when starting a new model from scratch and also correspond to a general office building.
  • The user can save settings for internal gains corresponding to other space types or other variations. These saved settings can be reloaded and applied to model other space types or to generate perturbations on the current model.
