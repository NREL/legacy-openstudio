Known Issues

  • Only Save your IDF file, do not save the SketchUp(skp) file. When you want to re open a project import the IDF file. Opening SKP files that have an associated IDF file can result in a set of duplicates surfaces that are slightly offset from the original surfaces
  • The Object Info Window doesn't take data until you tab to the next field or click elsewhere in the window.
  • Toolbar Tool tips may not work correctly on the Mac if you have made your Toolbars horizontal. The tips also don't' show on the Mac in the status bar under any conditions.
  • If you create a group out of multiple EnergyPlus Zones, and then hide one of those Zones you will get an error
  • You can not draw a sub-surface that shares two edges with a base surface. So you can put a window at the edge of wall, but not at an edge as well as the roof or floor.
  • You also can not place two sub-surfaces right against each other. Place a small space between them.
  • If you try and draw a shading surface in line with the roof it will mistakenly be recognized as a roof. If you do want a shading device in line with the roof you first have to draw it lower, and then move it up into place.
  • Shading surfaces also can not be draw while touching the floor. Similar to above, you would draw them smaller and then stretch them down the floor.This could occur on vertical fins
  • More_known_issues_to_add
  • Input fields in the Object Info window do not validate. It's possible to put in bad values.
  • The Object Info Window only works on one object at a time. If you select multiple objects or surfaces and make a change, it will only affect one of those selected objects.

Miscellaneous Notes

  • Added support for EnergyPlus 5.X
  • Added support for Windows 7
  • Removed dependence on wxSU
  • You can now run EnergyPlus Simulations as a non-admin user.
  • Zone Origins don't move once they are set. You can move the SketchUp group but it won't move the zone origin. If you want to move the Zone origin you need to edit the IDF file elsewhere. You can however set the zone rotation through the OpenStudio Object Info Window.
  • Creating a new zone or shading group requires the user to double-click the group to edit it instead of automatically opening for edit when the group is placed.

Troubleshooting

OpenStudio

  • We recognize that there are still bugs in OpenStudio. SketchUp is a flexible and dynamic program that allows the user to draw all kinds of geometry in many different ways...we certainly haven’t been able to test every single one of them. We welcome your assistance in identifying bugs so that we can get them fixed in the next version. We also welcome any feedback on how we might improve the plugin, feature requests, compliments, criticisms, etc.
  • For all bug reports, feedback, and questions about OpenStudio, please visit the collaborative project website on SourceForge.net.
  • The project website allows you to join the user support mailing list, browse the list of known issues in the bug tracker, or post a new bug report.
  • If the plugin detects that an error has occurred, an “Error Alert” window will appear. Your first step for resolving the error should be to visit the project website to check the bug tracker and user support mailing list to determine if someone else has encountered the same problem. If the issue has already been reported, there is a good chance you will find a workaround posted as well.
  • If you think you have found an unreported bug, the next step is to send an email to the user support mailing list. You can help us the most if you follow these suggestions:
    • Describe the bug in your email message. Tell us what you were doing when it happened, and if possible, describe the steps necessary to reproduce the problem.
    • If the “Error Alert” window appeared, copy and paste the error message and backtrace into your email.
    • Attach the EnergyPlus input file (.idf) and the SketchUp file (.skp) with your email.
    • We will try to respond to your email as quickly as possible. Thanks for your help and patience!
