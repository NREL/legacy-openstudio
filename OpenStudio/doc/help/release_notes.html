OpenStudio Documentation: Release Notes







Requirements

Supported Operating Systems

  • Windows XP
  • Windows Vista
  • Windows 7
  • Mac OS 10.x

Supported Versions of SketchUp

  • Google SketchUp 7.x Pro or free versions

Supported Versions of EnergyPlus

  • EnergyPlus 5.x

Installation and Setup

  • Download and install Google SketchUp 7
  • Download and install EnergyPlus 5.X
  • Download and install OpenStudio
(note: Admin rights will be required to install these applications)

New Features

  • Added support for EnergyPlus 5.0
  • Daylighting:Controls
  • Output:IlluminanceMap Input Objects
  • Surface and sub-surface search
  • Quickly add internal gains and HVACTemplate:Zone:IdealLoadsAirSystem to zones
  • Automatic surface matching and unmatching tool
  • Default construction manager
  • Render by Outside Boundary Condition
  • Improved documentation and tutorials

Bugs Fixed

  • Improved support for simple geometry
  • Removed dependency on Windows system dlls not shipped with plugin
  • Run EnergyPlus through OpenStudio as a non-admin
  • Back faces of EnergyPlus Shading Surfaces now have different colors than their front faces.
  • Stability improvements to the Object Info Dialog
  • Bugs fixed with patch to 1.0.5.348
    • Attached shading surfaces can now be manually assigned a base surface. By default the base surface is a floor in the zone, however you may need to change it to an exterior wall. EnergyPlus requires that shading surfaces have an exterior base surface.
    • When you click “Reset to Default Office” in Zone Loads “Constant Setponit Thermostat” is now set for HVACTemplase:Thermostat. This is only visible if you also turn on “Add HVAC Template:Aone:IdealLoadsAirSystem.”
    • When you save Zone Load Preferences, your Thermostat is also saved along with Zone Loads.
    • When you change your Outside Boundary Condition to something other than “Outdoors” sun and wind exposure are both properly turned off in the Object Text Window and the IDF file. If you change back to “Outdoors” then sun and wind exposure are both turned back on, they can be  manually turned back on.
    • Fixed issues  with conversion of Simple Objects to Detailed Objects.

Known Issues

  • When working with OpenStudio, save only your IDF file. Do not save the SketchUp SKP file. Opening SKP files that have an associated IDF file can result in a set of duplicates surfaces that are slightly offset from the original surfaces and cause other problems.
  • Toolbar Tool tips may not work correctly on a Mac if you have made your Toolbars horizontal. The tips also do not show on a Mac in the status bar under any conditions.
  • If you create a group out of multiple EnergyPlus Zones, and then hide a zone, you will receive an error message.
  • Rendering Settings Dialog does not work on a Mac because the JavaScript engine in Safari
  • Can't move zone origin through OpenStudio.
  • Surface Matching may not properly match some non rectangular surfaces. If this happens you can manually match the surfaces using the Object Information Window. An alternative workflow would be to draw them as rectangular surfaces, match them, and then adjsut the geometry
  • As much as possible you should avoid using curves (arcs or circles) to create your geometry as it will create a lot of faced surfaces and increase the simulation time. If you do use curves set the segment count as low as possible. Before you extrude a curve in SketchUp you can select it and right click to view the "Entity Information" window. There you can change the number of segments. If you want to make changes to curves via the object info window you have to "View Hidden Geometry" to select individual facets. You can't globally change the whole curve at once.
  • When you are drawing in a zone and extrude your plan up OpenStudio runs through and classifies all of the newly created base surfaces. Normally this is very quick, but if you have a complex shape, and also if you have the Object Information window and/or Outliner window open it make take longer. If you interrupt the process by exiting out of the zone before it is done, the zone won't function properly. If that happens, you can manually update the unclassified surfaces, or delete and redraw them.

Troubleshooting

OpenStudio

  • We recognize that OpenStudio is not perfect. SketchUp is a flexible and dynamic program that allows the user to draw all kinds of geometry in many ways, and we have not been able to test all of them. We welcome your assistance in identifying bugs so we can fix them in the next version. We welcome feedback about how we might improve the plugin. We would also like to learn about feature requests, compliments, criticisms, etc.
  • For all bug reports, feedback, and questions about OpenStudio, visit the collaborative project Web site at SourceForge.net.
  • The project Web site enables you to join the user support mailing list, browse the list of known issues in the bug tracker, or post a new bug report.
  • If the plugin detects an error, an Error Alert window appears. Your first step should be to visit the project Web site to check the bug tracker and user support mailing list to see if someone else has encountered the same problem. If the issue has already been reported, a workaround has likely been posted.
  • If you think you have found an unreported bug, the next step is to send an email to the user support mailing list. You can help us the most if you follow these suggestions:
    • Describe the bug. Tell us what you were doing when it happened, and if possible, describe how to reproduce the problem.
    • If the Error Alert window appeared, copy and paste the error message and backtrace it into your e-mail.
    • Attach the EnergyPlus input file (.idf) to your e-mail.
    • We will respond to your e-mail as quickly as possible. Thanks for your help and patience!
